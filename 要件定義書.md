# FishingLog Share 要件定義書

最終更新：2026-06-17（2回目）

---

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| アプリ名 | FishingLog Share |
| 種別 | シングルページアプリケーション（SPA）|
| 対象プラットフォーム | モバイルブラウザ（幅 430px 基準）|
| 使用技術 | HTML / CSS / JavaScript、Leaflet.js（地図）、OpenStreetMap Nominatim（ジオコーディング）|
| ファイル構成 | `index.html`（単一ファイル）|
| データ永続化 | `localStorage`（アカウント・セッション・投稿・グループ・プロフィール）|

### 2-2. 認証機能（ログイン・新規登録）

#### 認証画面
- アプリ起動時、未ログインの場合はアプリ全体を覆うオーバーレイ画面（`#auth-overlay`）を表示する
- 「ログイン」「新規登録」をタブで切り替えられる

#### 新規登録
- 入力項目：ニックネーム（20文字以内）・パスワード（6文字以上）・パスワード確認
- 登録時に **一意のユーザーID（`'u_' + Date.now()` 形式）** を内部で自動生成する
- 同じニックネームが既に存在しても登録可（ニックネームの重複を許可）
- パスワードは DJB2 ベースのハッシュ関数（`_hashPwd`）で変換して保存する（平文保存なし）
- 登録完了後は即座にログイン状態になりアプリに入る

#### ログイン
- 入力項目：ニックネーム・パスワード
- **ニックネームとパスワードハッシュを同時照合**（`accounts.find(a => a.name === name && a.passwordHash === _hashPwd(pwd))`）して認証する
- 同名ユーザーが複数いる場合もパスワードで一意に識別できる
- 認証失敗時は「ニックネームまたはパスワードが正しくありません」を表示

#### セッション管理
- ログイン成功時にセッション情報（id・name・emoji・photo）を `fishinglog_session` に保存する
- ページリロード時にセッションを復元し、セッションがあれば認証画面をスキップする
- `currentSession.id`（ユーザーID）を投稿・グループ・権限チェックの基準とする

#### ログアウト
- マイページの「🚪 ログアウト」ボタンから実行
- `fishinglog_session` を削除し、認証画面に戻る

### 2-3. マップ機能

#### 釣果ピン表示
- 投稿データの位置情報（lat/lng）を地図上にピンで描画する
- 同一ロケーションの投稿は 1 つのピンに集約する
- **ピンのデザイン**：白丸・青枠（`#1a73e8`）・直径 20px・ドロップシャドウ
- ピンをタップするとポップアップで魚種・場所・投稿者・時刻を表示する
- **自分の投稿のポップアップには「編集 📝」「削除 🗑️」ボタンを横並び表示**し、それぞれ編集モード遷移・削除確認モーダルを起動する

#### ウェザーカード（日付シード擬似データ）
- ページロード時および MAP タブ切り替え時に当日の「月/日(曜日)」を自動表示する
- 天気・気温・潮汐・風向・風速は **外部API不使用**、当日日付をシード値とした決定論的な擬似データを表示する
  - **ハッシュ関数**：`_dateRand(date)` → Knuth 乗算ハッシュで日付ごとに固定値を生成
  - **潮汐**：2024-01-11（大潮基準）からの経過日数を 30 日周期テーブルでルックアップ
  - **天気**：晴れ 40% / くもり時々晴れ 20% / 晴れのちくもり 18% / くもり 12% / 雨 7% / 雷雨 3% の重み付き選択
  - **気温**：月別基準気温 ± 3℃ の日次変動
  - **風向・風速**：8方位からランダム選択・2〜7 m/s
- リロードしても同日は必ず同じ値になる（純粋なランダムではなく日付依存）
- チェックインボタン押下時のポップアップ・トーストにも当日の擬似天気データを連動表示する

#### 釣果ガイドピン（⚓ 全国60大釣り場マスター）

##### 表示制御（ズーム連動）
- ズームレベル **10 未満**：ガイドピンを完全非表示（地図を軽量に保つ）
- ズームレベル **10 以上**：⚓ アイコンのガイドピンを表示し、港名と当日の狙い目魚種をラベルとして常時表示（`bindTooltip` の `permanent: true`）
- 地図のズーム（`zoomend`）・移動（`moveend`）イベントで可視状態を自動更新する

##### データ構成（`GUIDE_PORTS` 配列）
- 北海道〜沖縄まで全国 **60 箇所**の主要港・海釣り公園を収録（47都道府県カバー）
- 各地点に `id`・`lat`・`lng`・`name`・`pref` および `baseFish`（春夏秋冬 4 シーズン別魚種リスト）を定義
- 地域特性に合わせた魚種設定：北海道（コマイ・ホッケ・サケ）、東北（タラ・アイナメ）、東京湾（シーバス・タチウオ）、北陸（ブリ・フクラギ）、沖縄（タマン・ガーラ・グルクン）など

##### リアルタイム海況取得（Open-Meteo API 連携）
- ガイドピンが画面内に表示されると、各港の座標を使って 2 つの公式 API に非同期 fetch を実行する
  - **Marine API**：`https://marine-api.open-meteo.com/v1/marine` → 海水温（SST）・波高を取得
  - **Weather API**：`https://api.open-meteo.com/v1/forecast` → 風速・気温を取得
  - 両 API とも **CORS フリー・HTTPS・APIキー不要**。デプロイ後もブロックされない
- 取得した条件（SST・風速）と現在のシーズンをもとに `_selectActiveFish()` で活性魚種を選定する

##### 活性魚種選定ロジック（`_selectActiveFish`）
| 条件 | 加点対象 |
|---|---|
| SST > 22℃ | 温水系魚種（アジ・タチウオ・ハマチ等）+2点 |
| SST 15〜22℃ | 中間水温系（メバル・アジ等）+1点 |
| SST < 15℃ | 冷水系（カレイ・アイナメ等）+2点 |
| 風速 < 3 m/s | 表層系（アジ・シーバス等）+1点 |

上位 3 種を「今日の狙い目魚種」として tooltip・popup に表示する。

##### ビューポート内限定 API 通信（最適化）
- 60 箇所全体に一斉 fetch を行わず、`map.getBounds().contains()` で現在画面内の港のみを抽出した `visiblePorts` に絞って fetch する
- 30 分間キャッシュ（`GUIDE_CACHE_TTL`）により同じ港への重複取得を防ぐ
- 地図移動（`moveend`）時に新たに画面内に入った港のデータを自動取得・更新する

##### フォールバック設計（3 層）
1. **API 成功** → リアルタイム条件から選定。ラベルに魚種をそのまま表示
2. **API 失敗（catch）** → シーズン予測で上位 3 種。ラベルに「（※予測）」プレフィックスを付与
3. **キャッシュ Hit（30 分以内）** → 再取得せず既存データを即時反映（通信節約）

##### ポップアップ表示内容
- リアルタイム取得時：海水温・波高・風速・気温の現在値
- 今日の狙い目魚種（リアルタイム or シーズン予測）
- 春夏秋冬のシーズン魚種一覧

---

#### 投稿位置の微調整（ミニマップ）
- 釣り場名を入力後、「🔍 地図確認」ボタンで Nominatim API によるジオコーディングを実行する
- 結果をフォーム内のインラインミニマップ（高さ 180px）に表示し、ピンをドラッグまたは地図タップで位置を修正できる
- 「地図確認」を使用しない場合は投稿時に自動ジオコーディングのフォールバックを実行する

### 2-4. タイムライン機能

- 表示フィルター：全体 / グループ / 個人（`isMyPost` フラグで判定）
- 投稿カードの表示項目：アバター（写真または絵文字）、ユーザー名、投稿時刻、公開範囲、釣果画像（または魚絵文字）、魚種タグ、場所、いいね・コメント・地図で見るボタン
- **自分の投稿カードにのみ「✏️ 編集」「🗑️ 削除」ボタンを表示**する（ユーザーID による所有権チェック）

---

### 2-5. 投稿・編集・削除機能

#### 新規投稿フォーム
- 入力項目：写真（複数枚、最大9枚）、魚種（複数入力可）、釣り場、位置情報（ミニマップで微調整）、公開設定
- 公開範囲：全体公開 / グループ / 個人
- 投稿データには `accountId`（ユーザーID）を付与して保存する
- 投稿データは `posts` 配列に追加後、`localStorage` に即時保存する
- 投稿完了後はタイムラインタブに自動遷移する

#### 編集モード
- タイムライン、マイページ、マップポップアップの「✏️ 編集」ボタンから起動する
- フォームに既存データ（写真・魚種・釣り場・位置・公開設定）をすべて読み込む
- フォームタイトルが「投稿を編集 ✏️」に変わり、送信ボタンが「変更を保存する 💾」に変わる
- キャンセルボタンを表示し、タップすると元のタブに戻る
- 保存時は `posts` 配列の該当要素を上書きし `localStorage` を更新する

#### 削除機能
- タイムライン・マイページ・マップポップアップの「🗑️ 削除」ボタンから起動する
- **自分の投稿にのみ削除ボタンを表示**（`isMyPost` フラグで制御）
- 削除ボタン押下時にカスタム確認モーダル（`#delete-confirm-modal`）を表示する
- 確認後、`posts` 配列から対象投稿を削除し `localStorage` を更新する
- 削除後はタイムライン・マップピン・マイページを即座に再描画する

---

### 2-6. グループ機能

- グループ一覧の表示（アイコン絵文字・グループ名・メンバー数・参加コード）
- **グループ作成**：グループ名＋アイコン絵文字（12種から選択）を入力して新規作成。参加コード（6桁英数字）を自動発行する
- **グループ参加**：6桁の参加コードを入力して参加する
- **グループ編集**：詳細画面ヘッダーの「⚙️ 編集」ボタンからグループ名・アイコン絵文字を変更できる
- グループアイコン候補：🎣 🐟 🦑 🐙 🚣 🏄 ⛵ 🐠 🦈 🌊 🐡 🏕️
- グループデータは `groups` 配列で管理し `localStorage` に保存する
- 自分のメンバーエントリには `isMe: true` フラグを付与し、プロフィール変更時に一括更新する

---

### 2-7. プロフィール・マイページ機能

#### プロフィール編集（⚙️ プロフィール編集ボタン）
- 編集項目：ユーザー名・プロフィール写真（ファイルアップロード）・アバター絵文字
- アップロード画像は最大 240px にリサイズ（JPEG 0.82 品質）してから `localStorage` に保存する
- 写真未設定の場合はアバター絵文字を表示する
- 保存時に `posts`（`isMyPost: true`）・グループメンバー（`isMe: true`）の名前・絵文字を一括更新し、`fishinglog_session` および `fishinglog_accounts` も同期する

#### 統計ダッシュボード
- 釣行回数・総釣果数・魚種数を自分の投稿データから動的計算する（`updateStats()`）

#### 釣り場別ランキング
- 自分の投稿の釣り場名を集計し、上位5件を棒グラフで表示する（`renderPrefList()`）

#### 自分の投稿一覧
- `isMyPost: true` の投稿を一覧表示し、各カードから編集・削除できる

#### マイページ内ボタン一覧
| ボタン | 動作 |
|--------|------|
| ⚙️ プロフィール編集 | プロフィール編集モーダルを開く |
| 🚪 ログアウト | セッションをクリアしてログイン画面に戻る |
| 退会する（赤ボタン） | アカウント削除確認モーダルを開く |

---

### 2-8. アカウント削除（退会）機能

- マイページの「退会する」ボタン（赤背景・白文字）から起動する
- **二段階確認**：
  1. カスタムモーダル（`#delete-account-modal`）で「削除する（取り消し不可）」ボタンを表示
  2. ブラウザの `confirm()` で最終確認（「本当に戻せませんがよろしいですか？」）
- 削除実行時の連動処理：
  1. `accounts` 配列から対象アカウントを削除し `localStorage` を更新
  2. `posts` 配列から対象ユーザーの全投稿を削除（`accountId` 一致分 + `isMyPost:true` のレガシーデータ）し `localStorage` を更新
  3. `groups` 配列の全グループから `isMe: true` のメンバーエントリを削除し `localStorage` を更新
  4. セッションをクリアしてログイン画面に戻る

### 2-9. データ永続化

| キー | 内容 |
|------|------|
| `fishinglog_accounts` | アカウント配列（id・name・emoji・photo・passwordHash）|
| `fishinglog_session` | 現在のログインセッション（id・name・emoji・photo）|
| `fishinglog_profile` | プロフィール（名前・絵文字・写真 Base64）※セッションと同期 |
| `fishinglog_posts` | 投稿データ配列（写真含む・accountId含む）|
| `fishinglog_groups` | グループデータ配列 |

- データを変更するすべての操作の直後に `localStorage` を上書き保存する
- ページロード時に `localStorage` から各データを復元する
- セッション有無に応じて `myProfile` を初期化し、`refreshMyPostFlags()` で `isMyPost` フラグを再計算する

#### 投稿の所有権判定ロジック（`refreshMyPostFlags`）
- `post.accountId` が存在する場合：`currentSession.id === post.accountId` で判定
- `post.accountId` が存在しない場合（認証機能追加前のレガシーデータ）：既存の `isMyPost` フラグをそのまま維持

## 3. 非機能要件

| 項目 | 要件 |
|------|------|
| 対応デバイス | スマートフォン縦向き（430px 幅を想定）|
| 地図ライブラリ | Leaflet.js v1.9.4（CDN）|
| タイルレイヤー | OpenStreetMap |
| ジオコーディング | Nominatim API（`Accept-Language: ja`、位置確認・投稿時フォールバック用）|
| 天気データ | 外部API不使用・日付シード擬似データ（`file://` 環境でも動作）|
| 認証 | 外部API不使用・localStorage ベース・パスワードはDJB2ハッシュ保存 |
| ガイドピン海況 | Open-Meteo Marine / Weather API（CORSフリー・HTTPS・APIキー不要）|
| ネットワーク | 地図タイル・Nominatim・Open-Meteo API 呼び出し時のみ通信 |
